User Interaction

Centipede supports four ways of interacting with the user by default: getting four different types of values, and displaying a messagebox.

Asking For Input

All Ask for Input actions share some arguments, which have the same function in each:

Title
Displayed in the title bar of the form.
Prompt
Displayed at the top of the form, above the input boxes.

Text or Numeric

This action supports setting variables either to the exact text entered by the user, or treating the text as python expressions and evaluating it. Note that this can result in a type other than a string or a number, for instance, if the user enters [1,2,3], which will return a list. This behaviour is controlled by the Evaluate checkbox: if it is checked, input will be evaluated.

Variables takes a list of comma separated variable names, with optional labels. The syntax is variable1 : label for variable1, variable2 : label for variable2 (etc). The spaces after the commas and around the colons are optional, but recommended for readability.

True / False

This operates almost identically to Text or Numeric ask for input action, except it only allows the user to set True or False.

Multiple Choice

This requires a comma separated list of values to choose from, and will give you the index and/or the name of the selected option. The Radio Button checkbox toggles between displaying radio buttons or a dropdown menu.

File Browser

This displays a form with a single input box and a button to show the standard windows "open" dialog. Filter is used to control which file extensions are shown, and uses the syntax Text to display |*.extension. For instance, to only list Centipede job files, which have the extension .100p, you could use Centipede Jobs (*.100p) |*.100p.
