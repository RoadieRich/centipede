Running

You can run a job either by clicking on the run button, or by using the Run option in the Run menu.

Messages

As the job progresses, the actions will emit various messages, which will be displayed in the messages tab.

At the top of the tab are the filter buttons, which allow you to choose which message types to display - the message type is shown in the final column.

Stepping

Another option for running jobs is to step through it. In this mode, execution pauses after each action, giving you a chance to check that it has done what you expect. To continue with the next action, either select "Step" from the run menu again, or click the Run button, which will now be labeled "step".

Debugging

Debugging centipede jobs is a lot like solving problems in any other domain, and can be split into three steps:

  1. Identify the problem

    Determine why the expected behavour isn't happening. This will generally involve

    • Stepping through the job
    • Adding messagebox actions to check the value of variables
    • Comparing actions within centipede to those outside
    • Graphing the progam flow - are the branch actions doing what you expect

  2. Identify solutions

    What do you need to do to fix the problem identified above?

  3. Implement and test

    Make the change, and check it works correctly. If not, return to step 1.
