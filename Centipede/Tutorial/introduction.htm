Introduction

Centipede is a tool for automating repetitive computer based tasks, the sort of tasks that involve moving data from one piece of software, database or document to another.

For example: You can use Centipede to create a new document, starting with a document template and completing it using data from another source.

Centipede jobs contain actions, each action does one small and specific thing, and you can build jobs from any combination of actions. When a job is run, actions are processed in order. Flow control actions provide the possibility for looping and branching.

Data is stored within Centipede variables, and variables are available to all actions that follow.

The Centipede application window is divided into four panels:

  • The information panel, which displays the welcome message on startup, tutorials, or any html file assigned by the current Centipede job for providing information about the job
  • The action editor panel, which displays the actions that make up the current Centipede job
  • The action toolbox panel, which displays all available actions in categories
  • The output panel, which displays any messages generated by the actions, and the current value of any Centipede variables
